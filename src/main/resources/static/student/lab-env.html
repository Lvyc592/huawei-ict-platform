鸿蒙智慧台灯项目设备端开发指导手册

本实验将带您完成一个完整的鸿蒙智慧台灯项目，模拟使用 DevEco Studio 进行 HarmonyOS 设备端应用开发，集成 IoT 通信、传感器采集与智能控制逻辑。

实验目标

  • 掌握 HarmonyOS 应用工程结构与 Ability 生命周期
  • 熟悉 IoT 设备控制 API 的调用方式
  • 完成智慧台灯的开关 / 亮度 / 色温控制功能
  • 实现人体感应 + 光感自动控制逻辑

实验步骤

1创建工程

在 DevEco Studio 中选择 Application → Empty Ability 模板，工程名 SmartLamp，语言选择 ArkTS。

// 工程结构
SmartLamp/
├── entry/src/main/ets/
│   ├── pages/
│   │   └── Index.ets        // 主页面
│   ├── common/
│   │   └── LampController.ets   // 灯控逻辑
│   └── model/
│       └── DeviceModel.ets      // 设备模型
└── resources/

2编写主页面 Index.ets

在右侧编辑器中打开 Index.ets，可以看到已经为您预置了完整代码，请按以下注释完成逻辑补充：

import { LampController } from '../common/LampController'

@Entry
@Component
struct Index {
  private controller: LampController = new LampController()

  build() {
    Column() {
      Text('智慧台灯控制台')
        .fontSize(28)
        .fontWeight(FontWeight.Bold)

      Toggle({ type: ToggleType.Switch, isOn: this.controller.isOn() })
        .onChange((isOn: boolean) => {
          this.controller.toggle(isOn)
        })
    }
  }
}
提示：按 Ctrl + S 保存后系统会自动运行编译，并在终端输出运行结果。

3实现 LampController

点击左侧文件树中的 common/LampController.ets，完成开关、亮度、色温三个方法：

  • toggle(isOn: boolean) — 控制灯的开关
  • setBrightness(level: number) — 设置亮度 0~100
  • setColorTemp(kelvin: number) — 设置色温 2700~6500K

4编译并运行

点击右上角 ▶ 启动 按钮，编译将在底部终端中输出。运行后可在模拟器中查看台灯 UI 效果。

$ hvigorw clean
$ hvigorw assembleHap --mode module -p product default
# 构建成功！
$ hdc install entry/build/default/outputs/default/entry-default-signed.hap
# 安装完成：1/1

实验要求

  1. 独立完成至少 3 个方法的编码实现
  2. 在终端中执行 npm test 验证单元测试
  3. 实验报告以 PDF 格式上传至作业区

常见问题

Q1：编译报错 "Cannot find module @ohos.iothardware"？

A：在 oh-package.json5 中添加依赖 "@ohos.iothardware": "^1.0.0" 后同步即可。

Q2：模拟器无法启动？

A：点击右上角"重置环境"按钮，等待 5-10 秒后重试。
